# 在线考试系统 - 本地启动指南

## 环境要求

### 必须安装的软件

| 软件 | 版本要求 | 下载地址 |
|------|---------|---------|
| JDK | 1.8 | https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html |
| MySQL | 8.0.x | https://dev.mysql.com/downloads/installer/ |
| Maven | 3.9.x | https://maven.apache.org/download.cgi |
| Node.js | 18+ (推荐22.12.0) | https://nodejs.org/ |
| npm | 10.9.0 | 随Node.js安装 |

### 验证环境

```bash
# 检查Java版本
java -version
# 应显示 java version "1.8.0_xxx"

# 检查Maven版本
mvn -version

# 检查Node版本
node -v
# 应显示 v22.12.0 或更高

# 检查npm版本
npm -v
# 应显示 10.9.0 或更高

# 检查MySQL版本
mysql --version
```

## 快速启动步骤

### 第一步：初始化数据库

1. **启动MySQL服务**

   Windows: 在服务管理器中启动MySQL服务
   ```
   Win + R -> services.msc -> 找到MySQL -> 启动
   ```

2. **创建数据库**

   ```bash
   # 使用命令行登录MySQL
   mysql -u root -p
   
   # 输入密码: 123456
   
   # 执行初始化脚本
   source C:/path/to/online-exam-system/database/init.sql
   ```

   或者使用Navicat等图形化工具：
   - 连接MySQL (root/123456)
   - 右键点击数据库 -> 运行SQL文件
   - 选择 `database/init.sql`

3. **验证数据库创建成功**

   ```sql
   SHOW DATABASES;
   -- 应该能看到 online_exam
   
   USE online_exam;
   SHOW TABLES;
   -- 应该能看到7张表
   ```

---

### 第二步：启动后端服务

1. **进入后端目录**

   ```bash
   cd online-exam-system/backend
   ```

2. **修改数据库配置（如需要）**

   打开 `src/main/resources/application.yml`，修改以下配置：

   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/online_exam?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
       username: root
       password: 123456   # 修改为您的MySQL密码
   ```

3. **编译并运行**

   ```bash
   # 方式一：使用Maven命令（推荐）
   mvn clean install
   mvn spring-boot:run
   
   # 方式二：使用IDE（IntelliJ IDEA）
   # 1. 打开backend文件夹作为项目
   # 2. 等待Maven自动下载依赖
   # 3. 找到 OnlineExamSystemApplication.java
   # 4. 右键 -> Run 'OnlineExamSystemApplication'
   ```

4. **验证后端启动成功**

   看到以下输出表示启动成功：
   ```
   ========================================
     在线考试系统后端服务启动成功！
     接口地址: http://localhost:8080/api
   ========================================
   ```

   浏览器访问 http://localhost:8080/api/auth/login 应返回JSON格式的错误信息

---

### 第三步：启动前端服务

1. **进入前端目录**

   ```bash
   cd online-exam-system/frontend
   ```

2. **安装依赖**

   ```bash
   # 必须使用 --legacy-peer-deps 参数
   npm install --legacy-peer-deps
   ```

   如果安装失败，尝试以下方法：
   ```bash
   # 清除缓存
   npm cache clean --force
   
   # 删除node_modules重新安装
   rm -rf node_modules package-lock.json
   npm install --legacy-peer-deps
   ```

3. **启动开发服务器**

   ```bash
   npm run serve
   ```

4. **验证前端启动成功**

   看到以下输出表示启动成功：
   ```
   App running at:
   - Local:   http://localhost:3000/
   - Network: http://192.168.x.x:3000/
   ```

## 常见问题解决

### 问题1：后端启动报错 "Communications link failure"

**原因**：MySQL连接失败

**解决**：
1. 检查MySQL服务是否启动
2. 检查用户名密码是否正确
3. 检查MySQL端口是否为3306

### 问题2：后端启动报错 "Public Key Retrieval is not allowed"

**原因**：MySQL 8.0 连接配置问题

**解决**：已在application.yml中添加 `allowPublicKeyRetrieval=true`

### 问题3：npm install 报错 "ERESOLVE could not resolve"

**原因**：依赖版本冲突

**解决**：使用 `--legacy-peer-deps` 参数
```bash
npm install --legacy-peer-deps
```

### 问题4：前端页面空白，控制台报错

**原因**：可能是后端未启动或跨域配置问题

**解决**：
1. 确认后端服务已启动
2. 检查vue.config.js中的代理配置
3. 清除浏览器缓存

### 问题5：登录时提示 "用户名或密码错误"

**原因**：数据库未初始化或密码错误

**解决**：
1. 确认已执行init.sql
2. 检查数据库中是否有用户数据：
   ```sql
   SELECT * FROM online_exam.sys_user;
   ```

## 端口占用处理

如果8080或3000端口被占用：

### 后端端口修改

编辑 `src/main/resources/application.yml`：
```yaml
server:
  port: 8081  # 修改为其他端口
```

同时修改前端 `vue.config.js` 中的代理配置：
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8081',  // 修改为新的端口
    changeOrigin: true
  }
}
```

### 前端端口修改

编辑 `vue.config.js`：
```javascript
devServer: {
  port: 3001,  // 修改为其他端口
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

## 生产环境部署

### 后端打包

```bash
cd backend
mvn clean package -DskipTests
```

生成的jar包在 `target/online-exam-system-1.0.0.jar`

运行：
```bash
java -jar target/online-exam-system-1.0.0.jar
```

### 前端打包

```bash
cd frontend
npm run build
```

生成的静态文件在 `dist/` 目录，可部署到Nginx等Web服务器。
